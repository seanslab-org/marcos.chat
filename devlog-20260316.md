#### 2. OpenClaw WebSocket Protocol (reverse-engineered)
- Wait for `connect.challenge` event before sending connect
- Request format: `{type:"req", id:<UUID string>, method, params}`
- Response format: `{type:"res", id, ok:true/false, payload/error}`
- Connect params: `client.id:"webchat"`, auth token, operator scopes
- Session key from `payload.snapshot.sessionDefaults.mainSessionKey`
- Chat events: `event:"chat"`, `payload.state: "delta"|"final"`
- Message content: `[{type:"text", text:"..."}]` array format

### Protocol Discoveries (undocumented)
- Request frame `id` must be a non-empty string (UUID), not a number
- `client.id` must match a known constant (`"webchat"`, `"webchat-ui"`, `"cli"`, etc.)
- Gateway requires `auth.token` when `gateway.auth.mode` is `"token"`
- Scopes `["operator.admin","operator.write","operator.read"]` needed for chat.send
- Origin header must match `gateway.controlUi.allowedOrigins` — proxy must rewrite
- OpenClaw minimum context window: 16000 tokens

### Incidents

- Required physical power cycle; recovered after ~15 minutes
- Root cause: Q8_0 model (10GB) on 16GB unified memory left <200MB for OS
